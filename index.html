Methodology
📈

Pair-Power Combinatorics

Every rebalance, all signal pairs are scored by their joint-state predictive power on 90-day forward BTC returns (H90). A pair receives a power score based on the best joint state's hit rate and effective sample size.

  • Top 1% of pairs selected per rebalance
  • Significance floor: mean + 1σ of all pair scores
  • Minimum 5 pairs as fallback
  • Joint states: (+1,+1), (+1,-1), (-1,+1), (-1,-1), neutral combinations
🎯

2Y + 3Y Ensemble

Two independent combo signals are computed in parallel — one trained on 2-year rolling windows, one on 3-year windows — then averaged into a single ensemble composite.

  • Reduces sensitivity to training window length
  • 2Y window: more responsive to recent regime shifts
  • 3Y window: more stable pair rankings, less noise
  • Simple average of both combo signals
🏠

MVRV Regime Conditioning

Training data is soft-weighted by MVRV Z-score regime. Pairs trained on periods in the same market regime as today receive higher weight.

  • COLD: MVRV < 0 (undervalued)
  • NEUTRAL: 0 ≤ MVRV < 2 (fair value)
  • HOT: 2 ≤ MVRV < 5 (overvalued)
  • EXTREME: MVRV ≥ 5 (bubble territory)
  • Same regime → 1.0 • Adjacent → 0.5 • Distant → 0.1
⚙

Execution & Gate

Position sizing uses a discrete 10% step system with a Holm step-down statistical gate to filter low-confidence signals.

  • 5 adaptive OOS percentile thresholds (P17/P33/P50/P67/P83)
  • Each threshold crossed adds/removes 10% BTC exposure
  • Holm gate: only act when signal passes sequential binomial test (α=0.05)
  • 7-day directional cooldown to prevent whipsaw
  • Quarterly pair re-selection, daily signal reaction
🔄

Ternary Signal Scoring

All raw on-chain and macro signals are converted to ternary {+1, 0, -1} scores using 730-day rolling percentile windows before entering the combinatorics engine.

  • Raw value > 80th percentile → +1 (bearish)
  • Raw value < 20th percentile → -1 (bullish)
  • Otherwise → 0 (neutral)
  • Inverse-direction signals (VIX, Coinbase Prem, SP500 Trend) are sign-flipped
📊

Horizon Weights & Scoring

Pair selection uses H90 (90-day forward returns) which shows the strongest predictive signal. Inference scoring uses uniform weights across all 6 horizons.

  • Pair selection horizon: H90 (90 days)
  • Inference horizons: 3d, 7d, 14d, 30d, 90d, 180d
  • Uniform 1/6 weight per horizon (maximum entropy null)
  • Prior H30=0.30 weights were undocumented — uniform adopted on principle

Maximum entropy null hypothesis. Backtest difference not significant (p=0.30), but uniform adopted on principle.
2026-03-04 — D10 production version First production-ready strategy. 2Y+3Y ensemble, H90 pair selection, Holm gate, adaptive OOS thresholds, 7-day cooldown.
MABreak lookahead bias fix _find_events() in MABreakSignal now guards that close.index[fi] < before, preventing forward return leakage into training. Affects 6 of 55 signals (MABreak variants). Colleague's "fixed" file had this guard commented out — re-applied.
Activity filter moved inside training window (Fix 4) Signal activity filtering now runs per-window (not on full sample) to prevent candidate selection lookahead bias. Previously, signals with low full-sample activity were excluded globally, leaking future data into the signal universe decision.
Cooldown direction tracking fix (Fix 1, from Darko) apply_cooldown() now tracks last_attempted direction (not last_executed). v1 bug: blocked reversals left last_dir stale, allowing bypass of the cooldown on the next bar.
